# 月信产品需求文档

## 1. 文档信息

- 产品名称：月信
- 当前版本：P 版 MVP
- 文档日期：2026-06-26
- 产品形态：Web 可交互原型
- 后续目标平台：iOS / Android 移动端 App
- 核心定位：面向围绝经期女性的隐私优先健康记录与就诊沟通工具

## 2. 产品背景

35-45 岁女性进入围绝经期前后，常见身体变化不再只是“经期是否准时”，还包括周期提前或延后、跳月、点滴出血、潮热、夜汗、睡眠变差、情绪波动、脑雾等。这些变化往往私密、零散、持续时间长，用户很难在就诊时准确回忆发生时间、频率和严重程度。

现有女性健康产品多数围绕育龄期经期预测、备孕、社区内容或广告转化设计，对围绝经期用户的成熟感、隐私感和就诊沟通需求覆盖不足。月信的机会不是做一个更热闹的经期 App，而是做一个更克制、更可信、更适合长期记录的女性健康工具。

## 3. 产品目标

### 3.1 MVP 目标

1. 让用户在 20 秒内完成当天关键身体信号记录。
2. 帮助用户观察围绝经期相关变化，包括出血波动、潮热夜汗、睡眠、情绪和脑雾。
3. 自动整理近 30 / 90 天就诊沟通摘要，降低用户就诊前的信息整理成本。
4. 明确建立隐私信任：本地保存、零广告、不追踪、可导出、可清空。

### 3.2 非目标

1. 不做医疗诊断。
2. 不替代医生问诊、检查和治疗建议。
3. 不提供避孕或备孕决策建议。
4. P 版不做账号、云同步、社交社区和广告变现。
5. 不通过夸大风险制造焦虑。

## 4. 目标用户

### 4.1 核心用户

35-45 岁围绝经期女性。

用户特征：

- 对健康数据隐私敏感，不希望记录被上传或用于广告。
- 对广告、社区噪音、营销推荐容忍度低。
- 关注周期不规律、点滴出血、潮热、夜汗、睡眠、情绪和脑雾。
- 希望就诊时能清楚描述身体变化，而不是临时回忆。
- 对“AI 诊断”谨慎，更需要可解释、可核对的记录摘要。

### 4.2 次要用户

18-35 岁关注隐私安全的育龄女性。

用户特征：

- 需要基础经期记录和周期预测。
- 重视本地保存、隐私锁和快速隐藏。
- 可能用月信记录痛经、睡眠、心情、异常出血或就诊信息。

## 5. 核心问题与产品机会

| 用户问题 | 产品机会 |
| --- | --- |
| 身体变化零散，难以回忆 | 用低负担每日记录沉淀时间线 |
| 围绝经期不只关心经期 | 将潮热、夜汗、睡眠、脑雾、点滴出血纳入核心记录 |
| 就诊前整理信息困难 | 自动生成近 30 / 90 天就诊沟通摘要 |
| 担心隐私数据被上传或被广告利用 | 本地保存、零广告、不追踪、可导出、可清空 |
| 公共场合打开 App 尴尬 | 提供快速隐藏敏感信息和 PIN 隐私锁 |

## 6. 产品原则

1. 隐私优先：健康数据默认留在本机，不上传、不追踪。
2. 低负担：核心记录流程应尽量控制在 20 秒内。
3. 克制可信：使用“观察、趋势、沟通摘要”等表达，避免诊断式口吻。
4. 围绝经优先：默认体验服务围绝经期用户，而不是普通经期记录的换皮。
5. 零广告：不以广告、内容流、数据售卖作为核心商业路径。

## 7. MVP 功能范围

### 7.1 首次使用引导

用户首次进入产品时完成基础设置。

功能需求：

- 选择关注重点：围绝经健康 / 周期隐私。
- 填写最近一次月信开始日期。
- 填写平均周期和经期长度。
- 明确告知产品不上传记录、不做广告追踪。

验收标准：

- 保存后进入首页。
- 首页文案和记录重点根据关注重点切换。
- 用户可在“我的 - 周期设置”中重新设置。

### 7.2 首页

首页是每日打开后的主入口。

功能需求：

- 展示隐私承诺：本地保存、零广告、不追踪。
- 展示当前模式、周期阶段、距离下次月信预计天数。
- 展示今日提示。
- 展示潮热 / 夜汗、睡眠摘要。
- 提供 20 秒快速记录：潮热、夜汗、睡眠差、点滴出血。
- 提供“月信来了”快速标记。
- 提供出血观察入口。
- 提供隐藏敏感信息能力。

验收标准：

- 点击快速记录后立即保存。
- 首页摘要根据当天记录实时更新。
- 隐藏敏感信息后，首页不展示敏感健康摘要。

### 7.3 记录页

记录页用于补充完整的每日健康记录。

功能需求：

- 日历选择日期。
- 记录月信开始、点滴 / 异常出血、流量。
- 记录潮热、夜汗、睡眠、心情。
- 支持备注。
- 支持展开更多项目：疼痛、脑雾、亲密关系、保护方式。
- 支持复用昨天状态。
- 明确提示数据仅保存在本机。

验收标准：

- 每日记录按日期保存。
- 切换日期后可查看和编辑对应记录。
- 围绝经健康模式下，出血波动、潮热、夜汗、睡眠优先展示。

### 7.4 洞察页

洞察页把用户记录转化为趋势和沟通材料。

功能需求：

- 展示观察重点、平均周期、下次月信预测。
- 围绝经模式下展示异常出血观察。
- 展示症状负担：潮热夜汗、睡眠压力、脑雾。
- 周期隐私模式下展示痛经、心情、睡眠等身体信号。
- 展示就诊沟通摘要。
- 提供 30 天 / 90 天范围切换。

验收标准：

- 洞察基于本地记录生成。
- 文案明确说明仅作趋势参考，不代表医疗诊断。
- 没有足够记录时，应给出轻量、非焦虑的空状态提示。

### 7.5 就诊沟通摘要

就诊沟通摘要是 P 版 MVP 的核心价值功能。

功能需求：

- 支持近 30 天和近 90 天切换。
- 统计出血记录天数、点滴出血天数、潮热 / 夜汗天数、睡眠较差天数。
- 展示月信开始日期。
- 展示关键记录时间线。
- 支持一键复制摘要。
- 支持导出本地 HTML 报告。
- 复制受限时，提供可手动复制的文本。

验收标准：

- 摘要内容可以用于就诊前自我整理或就诊现场沟通。
- 导出报告可打开、打印或另存为 PDF。
- 报告包含隐私说明和非诊断说明。

### 7.6 我的页与隐私中心

我的页用于管理设置、隐私和数据控制。

功能需求：

- 展示当前模式和记录天数。
- 展示隐私承诺：数据仅保存在本机、无广告 SDK、不跨 App 追踪、不上传健康记录。
- 支持周期设置。
- 支持隐私模式开关。
- 支持每日记录提醒开关。
- 支持 4 位 PIN 原型锁。
- 支持立即锁定、清除 PIN、忘记 PIN 后清除本地锁。
- 支持复制就诊摘要、导出 HTML 报告、导出 JSON、清空本地数据。

验收标准：

- 设置 PIN 后可立即锁定。
- 正确输入 PIN 后解锁。
- 清空数据后记录和设置恢复默认状态。
- 用户能明确理解数据控制权在自己手里。

## 8. 核心用户流程

### 8.1 首次使用

1. 打开月信。
2. 选择关注重点。
3. 填写最近一次月信开始日期、平均周期、经期长度。
4. 进入首页查看预测和提示。

### 8.2 每日快速记录

1. 打开首页。
2. 查看今日提示和隐私状态。
3. 点击潮热、夜汗、睡眠差或点滴出血。
4. 数据保存到本机。
5. 首页和洞察同步更新。

### 8.3 完整记录

1. 进入记录页。
2. 选择日期。
3. 填写核心身体信号。
4. 展开更多项目补充记录。
5. 保存记录。

### 8.4 就诊前整理

1. 进入洞察页。
2. 选择近 30 天或近 90 天。
3. 查看关键指标和时间线。
4. 复制摘要或导出本地报告。
5. 就诊时使用摘要辅助沟通。

### 8.5 公共场合隐私保护

1. 在首页点击隐藏敏感信息。
2. 首页进入保护状态。
3. 需要查看时点击显示内容。
4. 需要更强保护时，在我的页开启 PIN 锁并立即锁定。

## 10. 隐私与安全要求

### 10.1 P 版要求

- 记录和设置保存于浏览器 localStorage。
- 不上传健康记录。
- 不接入广告 SDK。
- 不做跨 App 追踪。
- 支持导出 JSON 和 HTML 报告。
- 支持一键清空本地数据。
- 支持 PIN 原型锁和快速隐藏敏感信息。

### 10.2 移动端产品化要求

- 使用系统安全存储保存敏感设置。
- 使用本地加密数据库保存健康记录。
- 支持 Face ID / Touch ID。
- 支持系统通知提醒。
- 文件导出需明确由用户主动触发。
- 如引入云同步，必须提供明确授权、加密策略、删除机制和隐私政策。

## 11. 文案与内容规范

1. 使用“记录、观察、趋势、摘要、沟通”。
2. 避免使用“确诊、判断疾病、治疗建议”等诊断表达。
3. 异常出血相关提示应克制：提示值得留意，不制造恐慌。
4. 易孕窗口必须提示仅供参考，不能作为避孕依据。
5. 文案避免低龄化、可爱化、营销化。

## 12. 成功指标

### 12.1 P 版验证指标

- 首次设置完成率。
- 首页快速记录使用率。
- 连续 7 天记录率。
- 记录页保存率。
- 就诊摘要查看率。
- 摘要复制 / 导出率。
- 隐私中心查看率。
- 快速隐藏和 PIN 锁使用率。

### 12.2 用户访谈指标

- 用户是否理解产品不是医疗诊断。
- 用户是否相信数据没有上传。
- 用户是否愿意连续记录。
- 用户是否认为摘要能帮助就诊沟通。
- 用户是否觉得文案舒服、成熟、不冒犯。

## 13. 风险与注意事项

1. 医疗合规风险：不得将趋势观察表达为诊断结论。
2. 隐私信任风险：后续如加入云同步，必须谨慎解释数据流向。
3. 记录负担风险：记录项过多会降低连续使用意愿。
4. 商业化风险：广告会直接破坏核心定位。
5. 人群表达风险：不要把用户粗暴标签化，优先使用“关注重点”表达。

## 14. 后续迭代方向

### 14.1 用户测试版

- 邀请 5-10 位目标用户试用。
- 验证首页、记录、洞察、就诊摘要是否易懂。
- 重点访谈隐私信任、记录负担和就诊摘要价值。

### 14.2 围绝经专业度增强

- 细化出血波动观察规则。
- 分析潮热 / 夜汗高发时间。
- 统计睡眠受影响天数。
- 建立脑雾、情绪、睡眠之间的关联观察。
- 区分“自己看”和“给医生看”的摘要版本。

### 14.3 移动端产品化

- 迁移至 Expo / React Native。
- 接入系统通知。
- 接入生物识别锁。
- 使用本地加密数据库。
- 支持 PDF / 图片导出。

### 14.4 商业化探索

- 坚持零广告。
- 免费版保留基础记录、隐私保护和基础摘要。
- 付费版可探索长期趋势、更多报告模板、加密备份、多设备同步。
- 不以出售数据、广告推荐或私密内容流量化作为商业模式。

## 15. 当前结论

月信 P 版 MVP 已经具备完整闭环：首次设置、每日记录、趋势观察、就诊摘要、隐私保护和本地数据控制。下一阶段最值得做的不是继续堆功能，而是进入真实用户测试，验证围绝经定位、记录负担、摘要可用性和隐私信任是否成立。

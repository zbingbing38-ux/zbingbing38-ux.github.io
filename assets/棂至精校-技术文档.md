## 3. 前端核心流程

### 3.1 首页

文件：

- `miniprogram/pages/home/index.js`
- `miniprogram/pages/home/index.wxml`
- `miniprogram/pages/home/index.wxss`

职责：

- 选择文案单。
- 选择包装图片。
- 管理素材状态。
- 上传素材到云存储。
- 调用 `createReport` 云函数。
- 显示“假进度条”，降低等待焦虑。
- 报告生成后跳转报告页。
- 报告生成成功后清空首页素材，方便下一次校对。

文案单入口：

- `wx.chooseFile`：手机本地文件，若当前微信版本不支持则提示改用聊天记录。
- `wx.chooseMessageFile`：微信聊天记录文件。
- 手动输入/粘贴。

包装图入口：

- `wx.chooseMedia`：从相册选择。
- `pages/camera/index`：拍照。
- 选择后进入 `pages/subject/index` 做主体框选。

### 3.2 拍照页

文件：

- `miniprogram/pages/camera/index.js`

职责：

- 调用小程序相机组件。
- 拍照后进入主体框选页。
- 支持闪光灯切换。
- 支持相册入口。

### 3.3 主体框选页

文件：

- `miniprogram/pages/subject/index.js`

职责：

- 读取待处理图片。
- 按屏幕比例展示图片。
- 自动给出建议框选区域。
- 播放自动框选过程。
- 支持拖动、四边和四角自由调整。
- 用 canvas 裁切图片。
- 将裁切后的临时图片回传首页。

关键参数：

- 最小框选尺寸：`MIN_BOX_SIZE = 72`
- 最大输出边长：`MAX_CANVAS_SIZE = 1400`
- 输出格式：jpg
- 输出质量：0.92

### 3.4 报告页

文件：

- `miniprogram/pages/reports/index.js`

职责：

- 读取本地缓存 `latestReport`。
- 展示文案准确率、待确认数、条码状态。
- 展示报告列表。
- 支持报告项筛选。
- 支持点击报告项进入详情。
- 支持微信扫一扫验证条码。

条码验证：

```js

  onlyFromCamera: true,
  scanType: ["barCode"]
```

## 4. 云函数 createReport

文件：

- `cloudfunctions/createReport/index.js`

运行环境：

- Node.js 16.13
- 云函数超时：60 秒
- 内存：微信云开发控制台配置

依赖：

```json
{
  "wx-server-sdk": "~2.6.3",
  "pdf-parse": "^1.1.1",
  "mammoth": "^1.6.0",
  "xlsx": "^0.18.5"
}
```

职责：

1. 读取文案单内容。
2. 下载包装图片。
3. 调用百度 OCR 识别包装文字。
4. 调用百度条码识别接口尝试识别条码。
5. 规则比对包装 OCR 与标准文案。
6. 调用 DeepSeek 做语义复核和问题合并。
7. 生成报告。
8. 尝试写入云数据库 `reports` 集合。
9. 将报告返回前端。

## 5. 外部服务

### 5.1 百度 OCR

用途：

- 识别包装图片文字。
- 识别图片格式文案单。
- 尝试识别条码图形。

文字 OCR 接口：

```text
https://aip.baidubce.com/rest/2.0/ocr/v1/accurate_basic
```

条码/二维码识别接口：

```text
https://aip.baidubce.com/rest/2.0/ocr/v1/qrcode
```

Access Token 接口：

```text
https://aip.baidubce.com/oauth/2.0/token
```

云函数环境变量：

```text
BAIDU_API_KEY
BAIDU_SECRET_KEY
```

### 5.2 DeepSeek

用途：

- 理解文案单结构。
- 复核规则比对候选问题。
- 合并碎片化差异。
- 降级 OCR 噪声。
- 生成更接近人工校对语言的报告说明。

接口：

```text
https://api.deepseek.com/chat/completions
```

云函数环境变量：

```text
DEEPSEEK_API_KEY
DEEPSEEK_MODEL
```

默认模型：

```text
deepseek-v4-flash
```

## 9. 报告生成逻辑

当前实现分三层：

1. OCR 层  
   识别文字、条码，并保留部分位置和置信度信息。

2. 规则层  
   将标准文案拆成字段和片段，对包装 OCR 做相似度比对。

3. DeepSeek 层  
   对候选问题做语义复核、合并、降级或删除。

后续目标：

- 先理解文案单结构。
- 再判断当前校对对象。
- 最后按文案一致性分类输出报告。

目标分类：

- 文案有出入。
- 应出现但未识别到。
- 包装上多出的文案。
- 文案单中有但当前包装可不使用。
- 疑似 OCR 误识别。

## 10. 图示说明处理

问题：

包装背面可能有泵头/喷嘴/旋转等图示说明，位置靠近注意事项时，OCR 和 DeepSeek 容易误把它并入注意事项正文。

当前处理：

- OCR 识别结果保留位置。
- 按文字位置粗分 `left / center / right`。
- 命中“喷嘴、泵头、逆时针、顺时针、旋转、用手固定、无法向上旋起、拧紧”等词时标记为 `usage_diagram`。
- 给 DeepSeek 的输入中标记：
  - `category: 使用方法图示`
  - `blockType: usage_diagram`
  - `compareAs: usage_diagram_not_attention`
  - `note: 泵头/喷嘴/旋转图示说明，不要并入注意事项正文`

后续优化：

- 引入更通用的“辅助图示/旁注说明”分类。
- 判断图示附近短句、箭头说明、编号说明。
- 将图示说明独立成报告项，不参与正文缺失判断。

## 15. 已知问题和改进方向

当前已知问题：

- OCR 对曲面、小字、低清晰度图片仍可能误识别。
- DeepSeek 需要更稳定的文案组选择逻辑。
- 报告仍有旧版 `risk / pending / ok` 字段，需要逐步迁移到一致性分类。
- 百度条码识别不一定能识别所有包装图片中的条码，需要保留微信扫一扫。
- 本地 CLI 自动部署云函数偶尔会遇到微信云接口 `41002 system error`，可改用开发者工具手动部署。

优先改进：

1. 重构报告数据结构为 A-E 分类。
2. 增加当前校对对象确认。
3. 增加文案单结构化解析。
4. 优化图示说明/设计标注过滤。
5. 增加人工确认状态保存。

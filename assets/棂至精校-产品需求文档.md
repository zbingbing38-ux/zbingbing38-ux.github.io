# 棂至精校产品需求文档

版本：V1.0  
日期：2026-06-26  
产品定位：包装文案一致性校对小程序

## 1. 产品概述

棂至精校是一款面向包装设计、品牌、电商、生产和审核人员的包装文案校对工具。用户上传标准文案单和待校对包装图片后，系统通过 OCR 识别包装现有文案，再结合规则比对和语义复核，输出包装与文案单之间的具体出入。

产品的核心目标不是做法规风险判定，而是帮助用户快速发现“包装上实际出现的文字”和“标准文案单中应使用的文字”之间是否一致。

## 2. 目标用户

- 品牌方：确认包装稿是否符合最终文案单。
- 包装设计师：印刷前检查文字、规格、条码、使用方法、注意事项等是否漏改。
- 电商/运营人员：上架前快速检查包装图中的关键信息。
- 生产/打样人员：减少因文案错漏导致的返工。
- 小团队创业者：没有专职审核人员时，使用 AI 辅助做第一轮检查。

## 3. 用户痛点

- 包装文案信息密集，人工逐字核对耗时。
- 文案单常包含多个规格、多个版本、多个包装面，容易误把“不属于当前包装”的内容当成缺失。
- 包装图片中存在图标、箭头、设计标注、打样批注等非正文信息，容易干扰 OCR 和语义判断。
- 条码是否能扫出通常需要单独用手机验证，流程割裂。
- 传统校对结果太碎，不利于用户判断真正需要修改的地方。

## 4. 产品目标

首版目标：

- 支持上传文案单和包装图片。
- 识别包装图片中的文字内容。
- 比对包装现有文案与文案单之间的具体差异。
- 支持条码图形识别，并保留微信扫一扫人工验证入口。
- 生成清晰可读的校对报告。
- 支持用户重新开始下一次校对。

不做：

- 不校验条码数字对应的真实商品信息。
- 不做法规合规判定。
- 不替代人工最终确认。
- 不把所有文案单内容都默认要求出现在当前包装上。

## 5. 核心使用流程

1. 用户进入首页。
2. 上传标准文案单：
   - 手机本地文件。
   - 微信聊天记录文件。
   - 手动输入/粘贴文本。
3. 上传待校对包装：
   - 拍照上传。
   - 相册上传。
4. 系统进入主体文案框选页：
   - 自动给出建议框选区域。
   - 用户可拖动和自由调整比例。
   - 页面提示：“请框选主体文案，减少无关干扰”。
5. 用户确认后回到首页。
6. 点击“启动 AI 精校”。
7. 系统显示进度条，依次提示上传、识别、比对、整理报告。
8. 生成报告后进入报告页。
9. 用户查看：
   - 文案准确率。
   - 文案出入。
   - 应出现但未识别到。
   - 包装上多出的文案。
   - 疑似 OCR 误识别。
   - 条码识别/扫码状态。
10. 用户可点击报告项查看详情。
11. 用户可点击“扫一扫”验证实物条码。
12. 回到首页后，已完成的素材应清空，方便开始下一次校对。

## 6. 功能需求

### 6.1 文案单上传

支持格式：

- 手动输入文本。
- TXT / CSV。
- PDF。
- Word docx。
- Excel xlsx / xls。
- 图片格式：png / jpg / jpeg。

上传来源：

- 手机本地文件。
- 微信聊天记录。
- 手动粘贴。

要求：

- 用户选择文案单后，首页显示已准备状态。
- 支持重新选择文案单。
- 校对完成后，首页文案单状态清空。

### 6.2 包装图片上传

支持来源：

- 拍照。
- 相册。

要求：

- 首版最多支持 2 张包装图。
- 上传前进入主体框选页面。
- 保留原图路径和裁切图路径。
- 用户可以删除已选择图片。
- 校对完成后，首页包装图状态清空。

### 6.3 主体文案框选

目标：

减少背景文字、设计稿标注、图示说明、桌面杂物等无关信息干扰。

交互要求：

- 选完图片后直接进入框选页面，不弹出二次选择弹窗。
- 系统先展示一个自动框选过程。
- 默认框选主体包装文案区域。
- 用户可以拖动框。
- 用户可以自由比例调整框，不强制等比。
- 底部提示：“请框选主体文案，减少无关干扰”。
- 用户确认后返回首页。

### 6.4 OCR 识别

目标：

识别包装图片中的文字内容，为后续比对提供基础。

要求：

- 使用百度 OCR 识别文案单图片和包装图片。
- 包装 OCR 需要保留文字块位置、置信度和大致区域。
- 疑似低置信 OCR 结果不应直接作为主要错误。
- 图标旁说明、设计标注、条码旁辅助文字应尽量和正文分开。

### 6.5 文案单理解

系统需要先理解文案单结构，再判断当前包装适用哪些文案。

文案单可能包含：

- 多个规格，如 30ml / 75ml / 460ml。
- 多个条码。
- 正装 / 替换装 / 旅行装等版本。
- 正面 / 背面 / 盒身 / 标签等包装面。
- 使用方法、注意事项、成分、产品名称、生产信息等字段。

要求：

- 不把文案单里的全部内容都当作当前包装必须出现。
- 如果系统无法判断当前适用文案组，应提示用户人工确认。
- 对当前包装上已经识别到的内容，优先做一致性校对。

### 6.6 文案比对

报告分类不再按“低 / 中 / 高风险”作为核心逻辑，而按文案一致性分类：

A. 文案有出入  
包装上识别到的文案与当前适用文案不一致。

B. 应出现但未识别到  
当前适用文案应该出现，但包装 OCR 没有找到。

C. 包装上多出的文案  
包装 OCR 出现了内容，但当前适用文案中没有明确对应项。只提示，不直接判错。

D. 文案单中有，但当前包装可不使用  
例如其他规格、其他版本、其他包装面的文案。折叠显示，不作为缺失。

E. 疑似 OCR 误识别  
识别结果明显像 OCR 错误，建议人工确认。

### 6.7 DeepSeek 语义复核

DeepSeek 的定位：

- 帮助理解文案单结构。
- 帮助识别多个规格、版本、包装面、条码组。
- 帮助把碎片化差异合并成字段级问题。
- 帮助生成报告摘要。

DeepSeek 不做：

- 不作为唯一对错判定来源。
- 不删除原始规则比对证据。
- 不编造文案单或包装中不存在的内容。

### 6.8 图示和旁注处理

包装图片中可能出现图示说明，例如：

- 泵头/喷嘴打开方式。
- 箭头说明。
- 图标旁短文。
- 使用步骤图。
- 环保/回收/勿入眼等图标说明。

要求：

- 这类内容应优先归为“图示说明”或“辅助说明”。
- 不得自动并入相邻正文，例如把泵头图示文字并入注意事项。
- 如果需要比较，应作为独立字段处理。
- 无法确认时提示人工确认。

### 6.9 条码检测

首版策略：

- 后端尝试识别包装图片中的条码图形和编码内容。
- 报告展示条码状态。
- 保留微信扫一扫人工验证入口。

条码状态：

- 未检测。
- 已识别。
- 未识别。
- 无法正常扫码。
- 建议人工确认。

要求：

- 不校验条码对应商品信息。
- 不把条码识别失败作为阻塞报告生成的条件。
- 微信扫一扫只作为人工辅助验证。

### 6.10 报告页

报告顶部展示：

- 文案准确率。
- 待确认数量。
- 条码状态。
- 扫码验证入口。

报告列表支持筛选：

- 全部。
- 文案有出入/待确认。
- 条码。
- 字段。

报告详情展示：

- 问题标题。
- 标准文案。
- 包装识别。
- 判断说明。
- 建议动作。

## 7. 数据字段建议

报告项建议字段：

- `id`
- `category`
- `issueType`
- `field`
- `standardText`
- `ocrText`
- `matchedText`
- `imageSide`
- `suggestion`
- `manualStatus`
- `evidence`

`manualStatus` 支持：

- `unconfirmed`
- `confirmed_difference`
- `confirmed_ok`
- `ocr_error`
- `not_applicable`
- `ignored`

## 8. 非功能需求

性能：

- 单次报告生成建议控制在 60 秒内。
- 首版最多 2 张包装图，降低云函数超时风险。

安全：

- API Key 不写入前端代码。
- API Key 不上传 GitHub。
- `.env` 和本地私有配置必须忽略。
- 条码完整内容不写入公开日志。

稳定性：

- OCR 或 DeepSeek 失败时，尽量返回规则报告。
- 条码识别失败不阻塞报告生成。
- 云开发未配置时给出明确提示。

## 9. 当前已知限制

- OCR 对小字、反光、曲面瓶身、低清晰度图片仍可能识别不准。
- 多规格、多版本文案单仍需要进一步增强适用范围判断。
- 图示说明和正文的区分需要持续完善。
- 报告分类目前仍有部分旧字段，需要后续统一到新的文案一致性框架。
- 导出功能首版暂不开放。

## 10. 版本规划

V1.0：

- 文案单上传。
- 包装图片上传。
- 主体框选。
- 百度 OCR。
- 百度条码识别。
- 微信扫一扫验证。
- DeepSeek 语义复核。
- 基础报告页。

V1.1：

- 当前校对对象确认。
- 文案组/规格/版本/包装面选择。
- 报告分类改为 A-E 一致性分类。
- 图示说明、设计标注、正文文案更稳定拆分。

V1.2：

- 报告导出。
- 历史报告。
- 人工确认状态保存。
- 低置信 OCR 原图定位。

V2.0：

- 多包装面批量校对。
- 团队协作。
- 品牌文案库。
- 更强的主体识别和版面理解。
